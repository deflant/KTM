Knowledge Trust Manager (KTM) — Application Documentation

Overview

The Knowledge Trust Manager (KTM) is a ServiceNow application that provides a centralized workspace for monitoring, reviewing, approving, and maintaining knowledge articles sourced from multiple External Content Connectors (Confluence, SharePoint, Google Drive, Web Crawler, ServiceNow Docs, ServiceNow KB).

Core Value Proposition

  • Unified Knowledge Explorer — Single view of all articles from all connectors
  • Review Workflows — Structured approval process with reviewer assignments
  • Modification Request Lifecycle — Task-based change requests (extends task)
  • Freshness Scoring — Automatic aging tracking (Fresh → Aging → Stale)
  • Analytics Dashboards — KPIs, source health scorecards, reviewer performance
  • Source Exclusion Management — Configure which connectors to exclude from views
  • AI-Powered Triage — Automatic reviewer assignment via AI Agent
  • Approval Workflows — Resolution approvals via sysapproval_approver

AI Agent: KTM Article Triage Agent

An AI Agent that automatically triages newly created review records:

  1. Analyzes article metadata (title, source, freshness)
  2. Determines content domain
  3. Searches for available reviewers with workload balancing
  4. Assigns the best-fit reviewer and sets status to in_review

Trigger: Fires automatically on record creation (x_snc_knowledge_tr_review_record)

Security: Restricted to Coordinator and Admin roles

Cross-Scope Privileges

The application grants read access to External Content Connector tables:

  • sn_ext_conn_admin_connector_summary (connector summary)

  • sn_ext_conn_sn_docs_external_search_schema (ServiceNow Docs)
  • v_ais_external_search_content (AI Search virtual table)

Getting Started

Prerequisites

  • ServiceNow instance with External Content Connectors plugin installed
  • Knowledge Management (KM) plugin active
  • AI Search capabilities for external document sync

Accessing the Application

  1. Navigate to Knowledge Trust Manager → Workspace in the application navigator
  2. The React SPA workspace opens with the Home dashboard
  3. Use the sidebar navigation to access different sections

Role Assignment

  • Assign x_snc_knowledge_tr.admin for full administrative access
  • Assign x_snc_knowledge_tr.coordinator for assignment management and analytics
  • Assign x_snc_knowledge_tr.reviewer for article review and modification requests
  • Assign x_snc_knowledge_tr.consumer for read-only access

Key Workflows

Article Review Process

  1. Articles are synced from external connectors or ServiceNow KB
  2. AI Triage Agent automatically assigns a reviewer
  3. Reviewer examines the article and either approves (with notes) or creates a modification request
  4. Approved articles are marked as reviewed; modification requests enter the task lifecycle

Modification Request Lifecycle

  1. Reviewer creates a modification request linked to a review record
  2. Review record status changes to "modification_requested"
  3. Assigned user works on the modification (Draft → Submitted → In Progress)
  4. Upon resolution, a resolution summary is required
  5. An approval is automatically created for System Admin validation
  6. Review record is set to "re_review_needed" for re-examination

Freshness Scoring

  • Fresh (0-30 days) — Article is current
  • Aging (31-89 days) — Article may need attention
  • Stale (90+ days) — Article needs immediate review
